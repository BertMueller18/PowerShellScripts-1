<h2 style="text-align: left; padding-left: 30px;"><a id="What is full mailbox access?"></a>What is full mailbox access?</h2>
<p>By granting full access mailbox rights to certain user on someone else’s mailbox, that user will be able to see all mailbox items (emails, contacts, calendars, notes, tasks) of that user, both in inbox and sent items, but he will not be able to send emails as or in behalf of that user.</p>
<p>Full mailbox access can be set from Exchange Management Console, Exchange Control Panel (ECP) or Exchange Management Shell (PowerShell) - depending on the platform for which it is being set.</p>
<p>Our recommendation would be to create a dedicated user for this migration which will have all necessary permissions (e.g. <a href="mailto:migration@someorganization.com">migration@someorganization.com</a>) and use that account. When the migration is completed simply delete it and with it all permissions which were assigned to it will be removed as well.</p>
